# 팀별 역할 가이드 & 미팅 보고 체계

> 이 문서는 각 팀의 업무 범위, 리드와 팀원의 역할 경계, 성장 방향을 정리한 가이드입니다.
> "이건 내가 해야 하는 일인가, 팀원이 해야 하는 일인가?"가 명확해야 팀이 빠르게 움직입니다.

---

## 우리 팀의 원칙

### 리드의 역할은 "설계 + 의사결정 + 코칭"이다
리드는 직접 모든 일을 하는 사람이 아닙니다. **방향을 설계하고, 핵심 의사결정을 하고, 팀원이 성장하도록 코칭하는 것**이 리드의 일입니다. 동시에 우리는 작은 팀이기 때문에 리드도 직접 고객을 만나고 딜을 클로징합니다. **관리 50% + 실무 50%**가 우리 팀 리드의 현실적 비율입니다.

### 팀원의 역할은 "실행 + 보고 + 성장"이다
팀원은 리드가 설계한 방향 위에서 **구체적인 실행을 책임지고, 결과를 보고하고, 매 업무에서 한 단계씩 성장하는 것**이 역할입니다. 시킨 것만 하는 게 아니라, 실행하면서 발견한 것을 리드에게 공유하고 더 나은 방법을 제안하는 것까지가 팀원의 역할입니다.

### 업무 경계가 명확해야 협업이 빠르다
"이건 누가 해야 하는 거야?"가 나오는 순간 속도가 느려집니다. 이 문서에서 각 팀별로 **리드가 하는 일 / 팀원이 하는 일 / 함께 하는 일**을 구분해두었습니다. 애매한 건 리드가 판단합니다.

## 공통 — 미팅 보고 규칙

- 미팅 후 **24시간 이내** Slack 팀 채널에 보고서 게시
- 제목 형식: **`미팅 보고_YYYY.MM.DD_대상명`**
- 본인이 직접 판단한 부분은 반드시 **굵게(bold)** 표시
- Follow-up 항목에는 반드시 **담당자 + 마감일** 명시
- 승인되면 **이틀 내 고객에게 연락**, 한 고객을 **최소 3번** 만나는 것이 기본

## 공통 — AI 활용 원칙

AI는 우리 모두가 활용하는 도구입니다. 다만 **도구를 쓰되, 판단은 본인이 해야 합니다.** 아래 원칙은 리드/팀원 관계없이 전원에게 적용됩니다.

| 원칙 | 구체적 실행 |
|------|------------|
| **AI 결과물 그대로 제출하지 않기** | 모든 제출물에 **"본인이 수정/보완한 부분"**을 bold로 표시해주세요 |
| **기획/전략 문서는 본인이 먼저** | 뼈대(방향, 핵심 메시지, 구조)는 본인이 먼저 잡고, AI는 살붙이기 용도로 활용 |
| **판단 근거는 본인이 설명** | "AI가 이렇게 말했으니까"는 근거가 아닙니다. 왜 이 방향인지 본인의 판단을 설명해주세요 |
| **고객/파트너 대면 자료는 3단계** | AI 초안 → 본인 수정 → 리드 확인을 거친 후 발송 |

> AI를 잘 활용하는 것도 역량입니다. 핵심은 "AI가 만든 것"과 "내가 판단한 것"이 구분되는 것입니다.

# 1. 국내 K-12 코들 세일즈팀

**채널**: `#고객_국내`
**미션**: 초/중/고등학교에 코들을 판매하고, 도입 후 활용까지 책임진다.

**팀 업무 범위**:
코들 세일즈 · 선생님 관계 관리 · 동아리 운영 지원 · 코들 활용법 안내 · 선생님 연수

### 팀 KPI — 매출 확대

| KPI | 측정 기준 | 리드 책임 | 팀원 책임 |
|-----|----------|:---------:|:---------:|
| **신규 학교 도입 수** | 분기별 신규 계약 학교 수 | 타겟 선정 + 클로징 | 후속 실행 + 데모 세팅 |
| **기존 학교 재구매율** | 전년 대비 재계약 비율 | 핵심 관계 유지 | 일상 관리 + 활용 지원 |
| **파이프라인 전환율** | 컨택 → 미팅 → 제안 → 클로징 각 단계 전환율 | 전체 파이프라인 관리 | 데이터 입력 + 업데이트 |
| **매출액** | 분기/반기/연간 코들 라이선스 매출 | 딜 설계 + 가격 협상 | 견적서 전달 + 계약 서류 |

> 리드는 파이프라인 숫자를 매주 파악하고, 이 KPI와 연결해서 "이번 달 어디가 부족한지" 직접 판단해야 합니다.

## 박기남 — 리드

> "학교 현장을 가장 잘 아는 사람이 숫자로 관리하면, 가장 강력한 세일즈 리더가 된다."

### 리드가 직접 해야 하는 일

**1. 세일즈 전략 & 의사결정**

| 업무 | 구체적 내용 | 주기 |
|------|------------|------|

| 세일즈 전략 수립 | 학교별 공략 포인트 설계. "이 학교는 교장 결재 vs 담당교사 결재" 의사결정 구조 파악 | 신규 학교마다 |

| 파이프라인 관리 | 전체 학교를 컨택 → 미팅 → 제안 → 클로징 단계로 분류하고 숫자로 관리 | 매주 업데이트 |

**2. 고객 대면 & 관계 구축**

| 업무 | 구체적 내용 | 주기 |
|------|------------|------|
| 핵심 미팅 직접 참석 | 의사결정자(교장/교감/담당교사) 미팅은 반드시 리드가 참석 | 건별 |

| 선생님 연수 기획 | 연수 목적/구성/일정 설계. 어떤 내용을 어떤 순서로 전달할지 뼈대를 잡는 것 | 연수건별 |
| 핵심 선생님 관계 관리 | 학교별 키맨(핵심 담당 선생님) 직접 관리. 정기적 안부 + 니즈 파악 | 월 1회 이상 |

**3. 팀원 코칭 & 관리**

| 업무 | 구체적 내용 | 주기 |
|------|------------|------|
| 이주현 님 일일 업무 지시 | "오늘 할 일 3가지 + 마감 시간 + 산출물 형태"로 구체적 지시 | 매일 |
| 완료 보고 확인 | 이주현 님의 퇴근 전 보고를 매일 확인하고 빠진 것 피드백 | 매일 |
| 세일즈 스킬 코칭 | 이주현 님이 고객 연락/미팅에서 성장할 수 있도록 실전 피드백 | 주 1회 |
| 미팅 보고서 검토 | 이주현 님이 작성한 보고서의 품질 확인 후 게시 승인 | 건별 |

**4. 상위 보고**

| 업무 | 구체적 내용 | 주기 |
|------|------------|------|
| 파이프라인 현황 보고 | 컨택/미팅/제안/클로징 단계별 건수 + 이번 주 핵심 이슈 | 주 1회 (금요일) |
| 딜 이슈 즉시 보고 | 클로징 위험, 경쟁사 이슈, 예산 변동 등은 발생 즉시 대표에게 공유 | 즉시 |

| 데모 세팅 및 진행 | 학교 시범 수업/데모를 직접 기획하고 진행 | 데모 3일 전 완료 |

**2. 선생님 관계 관리 — 도입 후 관리가 재구매를 만든다**

|------|------------|------|

| 동아리 운영 지원 | 코들 활용 동아리에 필요한 자료/계정/커리큘럼 전달 | 학기 초 + 수시 |
| 연수 기획 참여 + 실행 | 박기남 님과 함께 연수 기획에 참여. 현장 실행은 직접 리드 | 연수건별 |
| 선생님 정기 연락 | 도입 학교 선생님들에게 안부 + 추가 니즈 파악 + 재구매 기회 발굴 | 월 1회 |

**3. 데이터 관리 & 보고**

| 업무 | 구체적 내용 | 주기 |
|------|------------|------|

## 성장 로드맵

**박기남 님 — 리드 성장 방향**
- 현재: 학교 관계 구축에 강함. 세일즈 프로세스를 숫자로 구조화하는 단계
- 6개월 목표: 파이프라인을 숫자로 독립 운영. "이번 달 컨택 00건, 미팅 00건, 클로징 00건" 자연스럽게 보고
- 1년 목표: 이주현 님을 독립적으로 성장시킨 경험을 갖춘 리더

**이주현 님 — 팀원 성장 방향 (3년차)**
- 현재: 재구매/클로징/파이프라인 관리를 이미 하고 있음. 이것을 체계적으로 루틴화하는 것이 핵심
- 6개월 목표: 본인 담당 학교 20곳을 체계적으로 관리. 학교별 상태/다음 액션/연락 주기가 정리된 루틴 확립
- 1년 목표: 20곳 관리를 안정적으로 운영하면서 신규 학교까지 확장. 박기남 님과 학교를 나눠서 각자 독립 관리하는 구조

사업 수주 · 대학교 관리 · 교수님 관계 관리 · 파일럿 운영 · 대학 연계 사업 수주

**팀 업무 범위**:
사업 수주 · 네트워크 관리 · 교원 연수 사업 · K-12 사업운영 (SW동행프로젝트, 디지털새싹 등)

### 팀 KPI — 국제학교 고객수 확대

| KPI | 측정 기준 | 리드 책임 | 팀원 책임 |
|-----|----------|:---------:|:---------:|
| **국제학교 신규 도입 수** | 분기별 코들 신규 도입 국제학교 수 | 타겟 학교 선정 + 파트너 미팅 + 클로징 | 학교 리서치 + 자료 번역 + 후속 연락 |

# 전체 요약 — 리드 vs 팀원 역할 구분 원칙

| 구분 | 리드가 한다 | 팀원이 한다 |
|------|------------|------------|
| **전략** | 방향 설계, 타겟 선정, 공략 전략 수립 | 리서치 자료 수집, 데이터 정리 |
| **의사결정** | 수주 여부, 가격, 파트너 구성, 참여 판단 | - (의견 제시는 환영) |
| **고객 대면** | 의사결정자 미팅, 핵심 관계 관리, 클로징 | 동행/기록, 후속 연락, 일상 관리 |
| **제안서** | 뼈대 설계 (메시지/목차/차별점) | 데이터 수집, 초안 작성, 자료 정리 |
| **사업 설계** | 운영 프로세스, 품질 기준, 투입 계획 | 현장 실행, 데이터 수집 |
| **팀원 관리** | 업무 지시, 코칭, 보고 확인, 성장 지원 | - |
| **보고** | 대표에게 팀 통합 보고 | 리드에게 실행 결과 보고 |
| **실행** | 핵심 딜 직접 실행 (관리 50% + 실무 50%) | 구체적 실행 전담 |
| **성장** | 팀원이 성장하도록 코칭 | 매 업무에서 한 단계씩 성장 |

# Justin 피드백 안내

미팅 보고서를 Slack에 올리면 Justin이 아래 형식으로 피드백을 드립니다.

### 승인 시

```
📋 Justin 피드백 | [고객사명] YYYY.MM.DD | 작성자: OOO

보완할 점:
- (다음부터 이렇게 해주세요)

잘한 점:
- (이번에 잘하신 부분)

다음 액션:
- 📩 연락 (이틀 내): 고객에게 보낼 연락 안내
- 📝 연락 문구 (안): 이메일 초안 (반드시 본인 검토 후 발송)
- 🔄 Follow-up 방법: 회신 있을 때 / 없을 때 대응법
- 🤝 3회 미팅 플랜: 1차 → 2차 → 3차 미팅 목표와 준비물

→ ✅ 승인 / ⚠️ 승인 (보완 요청)
```

### 반려 시

```
📋 Justin 피드백 | [고객사명] YYYY.MM.DD | 작성자: OOO

보완할 점:
- (이 부분을 수정해서 재제출해주세요)

잘한 점:
- (이번에 잘하신 부분)

→ ❌ 반려 (사유)
```

> 반려 시에는 다음 액션 가이드가 나오지 않습니다. 보고서를 보완해서 재제출하시면 승인과 함께 다음 액션을 안내해드립니다.

### 반려 기준

| 사유 | 예시 |
|------|------|
| 필수 5개 항목 중 1개 이상 통째로 누락 | Follow-up 섹션 자체가 없음 |
| Follow-up 전체에 담당자/마감일이 하나도 없음 | "추후 논의", "확인 예정"만 있고 구체적 일정 없음 |
| 미팅 내용 전체가 사실 나열만 | "관심 보임", "공감함", "좋은 미팅이었다" 수준 |

# 꼭 기억해주세요

1. **보고 없는 미팅은 안 한 것과 같습니다.** 미팅 후 24시간 내 보고서를 올려주세요.
2. **본인의 생각이 보고서에 담겨야 합니다.** AI가 정리한 것과 내가 판단한 것은 다릅니다. 본인 판단은 **bold**로 표시해주세요.
3. **"연락 기다리겠습니다"는 금지입니다.** 우리가 먼저 다음 일정을 잡고, 먼저 자료를 보내고, 먼저 제안합니다.
4. **한 고객을 최소 3번 만나세요.** 1차에서 니즈 파악 → 2차에서 시범/제안 → 3차에서 클로징. 이 흐름을 만드는 게 세일즈입니다.
5. **막히면 30분 이상 혼자 끙끙대지 마세요.** 바로 공유하고 상의하세요. 그게 더 빠릅니다.
6. **리드는 "설계 + 의사결정 + 코칭"이다.** 직접 다 하려고 하지 마세요. 팀원이 성장하도록 시키고 확인하는 것이 리드의 일입니다.
7. **팀원은 "실행 + 보고 + 성장"이다.** 시킨 것만 하는 게 아닙니다. 더 나은 방법을 발견하면 제안해주세요.
